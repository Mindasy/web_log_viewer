# 使用实例

## 实例一：分析 Bracket Log 格式日志

Bracket Log（括号格式）是一种常见的结构化日志格式，典型行如下：

```
[2026-01-15 08:00:00,000 +0800][INFO][1234][567][WEB-API][com.example.web.UserController] 用户 admin 登录成功，IP: 192.168.1.100
```

### 操作步骤

1. 点击「📂 打开」或拖放日志文件到浏览器窗口
2. 解析向导自动弹出
3. 在「预设格式」中选择 **Bracket Log（括号格式）**
4. 观察「测试结果」区域的字段映射是否正确

   | 字段 | 提取值 |
   |------|--------|
   | timestamp | `2026-01-15 08:00:00,000 +0800` |
   | level | `INFO` |
   | pid | `1234` |
   | tid | `567` |
   | tag | `WEB-API` |
   | source | `com.example.web.UserController` |
   | message | `用户 admin 登录成功，IP: 192.168.1.100` |

5. 点击「应用」完成解析

### 后续分析

- 勾选表格上方的级别标签切换不同级别可见性
- 在搜索框输入 `ERROR` 搜索所有错误
- 切换到「智能识别」面板查看分词结果
- 打开「统计」面板查看级别分布

## 实例二：使用智能规则生成器解析自定义格式

假设你的日志格式如：

```
15/Jan/2026:08:12:33 +0800 [ERROR] web-01 myapp: 数据库连接超时
```

### 操作步骤

1. 加载文件，解析向导弹出
2. 选择「智能识别」标签
3. 系统自动对样本行分词，显示颜色编码的 token
4. 点击每个 token 分配字段类型：

   | Token | 建议字段 |
   |-------|---------|
   | `15/Jan/2026:08:12:33 +0800` | timestamp（点击直到切换为 timestamp） |
   | `[ERROR]` | level |
   | `web-01` | source（或忽略） |
   | `myapp` | source |
   | `:` | separator（分隔符） |
   | `数据库连接超时` | message |

5. 观察「测试结果」区域实时确认匹配结果
6. 点击「应用」完成解析
7. 解析完成后，系统会询问是否保存此 Pattern（方便下次复用）

## 实例三：搜索与过滤

### 基本搜索

- 在搜索框输入 `NullPointerException`，所有包含该文本的行立即过滤
- 开启「Aa」（大小写敏感）搜索 `null` 不匹配 `NullPointerException`
- 开启「W」（全词匹配）搜索 `getUser` 不匹配 `getUserById`
- 开启「.*」（正则模式）搜索 `error|fail|timeout` 同时匹配多种关键词

### 搜索结果导航

- 按 `F3` 跳转到下一个匹配行
- 按 `Shift+F3` 跳转到上一个匹配行
- 在跳转输入框输入 `#3` 跳转到第 3 个匹配结果

### 级别过滤

点击级别标签可以切换日志级别的可见性：

- 取消勾选 `DEBUG` 和 `TRACE`，只查看 INFO 及以上级别的日志
- 勾选 `FATAL` 和 `ERROR`，聚焦于错误日志

### 高级过滤

点击「⚙️ 高级过滤」展开过滤栏：

- **线程过滤** — 输入 `http-nio` 只查看 HTTP 请求线程的日志
- **来源过滤** — 输入 `UserController` 只查看特定类的日志
- **消息过滤** — 输入 `超时` 只查看超时相关日志
- **时间范围** — 选择起止时间，只查看该时间窗口内的日志

## 实例四：使用压缩包

### ZIP 压缩包

```bash
# 生成测试 ZIP 文件
python3 scripts/generate_archives.py

# 输出 example/test_log.zip（内含 test_log.log）
```

1. 将 `test_log.zip` 拖入浏览器窗口
2. 工具自动解压并找到其中的 `.log` 文件
3. 左侧文件列表面板显示 `test_log.zip / test_log.log`
4. 文件名前显示 `ZIP` 标签

### tar.gz 压缩包

同样支持 `test_log.tar.gz`、`test_log.tgz`、`test_log.tar`、`test_log.log.gz` 等多种格式。

### 多文件合并

如果压缩包内含多个日志文件，所有文件都会解析并合并到一个视图中：

- 左侧文件列表按压缩包分组显示
- 点击文件名仅显示该文件的行
- 点击「🔗 合并」可添加更多文件（支持混合压缩包和普通文件）

## 实例五：批量生成测试数据

项目提供了 Python 测试数据生成脚本：

```bash
# 生成 20 个 bracket log 格式的 ZIP 文件（每个 10 万行）
cd scripts
python3 generate_samples.py
# 输出 example/app_1.zip ~ example/app_20.zip

# 生成多种压缩格式测试文件
python3 generate_archives.py
# 输出 example/test_log.zip, test_log.tar.gz, test_log.tgz, test_log.tar, test_log.log.gz
```

## 实例七：Pattern 管理

### 保存解析规则

1. 配置好解析规则（自定义正则或智能识别）
2. 点击「保存」按钮
3. 输入 Pattern 名称和描述
4. 下次打开相同格式的日志时，直接在预设中选择即可

### 导出与导入

1. 打开 Pattern 管理器（工具栏「📋 Pattern」）
2. 点击「导出」按钮保存为 JSON 文件
3. 另一台电脑上打开 Pattern 管理器，点击「导入」上传 JSON 文件

## 实例八：书签与跳转

### 添加书签

- 双击日志行添加/删除书签（🔖 列会显示书签图标）
- 或选中后按 `Ctrl+B`

### 管理书签

1. 点击「📋 书签列表」打开书签面板
2. 列表显示所有书签的时间戳和消息摘要
3. 点击书签跳转到对应行
4. 点击「🗑️ 清除所有」清空书签
5. 点击「💾 导出」保存书签

### 跳转技巧

| 输入 | 效果 | 示例 |
|------|------|------|
| `42` | 跳转到第 42 行 | `42` |
| `#3` | 跳转到第 3 个搜索结果 | `#3` |
| `@2` | 跳转到第 2 个书签 | `@2` |
| `$` | 跳转到末行 | `$` |
| `+100` | 从当前行向后 100 行 | `+100` |
| `-50` | 从当前行向前 50 行 | `-50` |
